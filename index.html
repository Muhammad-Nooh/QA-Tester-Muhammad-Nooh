Designed High-Impact Scenarios

1. Negative Logic: Interest Reversal

Verifying that reversing a transaction correctly restores the interest accrual state without affecting subsequent payments.

2. The "Race Condition" Test

Simulating two simultaneous API calls for a single loan payoff to ensure only one transaction completes.

3. Cross-Module Integrity

Verifying that a change in "Customer CRM" status (e.g., Deceased) automatically freezes linked Deposit accounts.

4. Boundary: 100M SYP Threshold

Testing batch opening logic precisely at the maximum system limit to ensure no buffer overflow or logic crashes.

5. UI Responsiveness: Data Tables

Testing 1000+ transaction rows on mobile viewport to ensure vertical/horizontal scrolling doesn't break the UI.

6. API Payload Tampering

Using Postman to intercept a repayment request and changing the amount to a negative value to test server-side rejection.

7. Late Fee Grace Period

Validating that the system does not apply a late fee if payment is received exactly 1 second before the grace period ends.

8. Multi-Currency Ledgering

Verifying exchange rate conversion accuracy when a loan in USD is repaid via a local SYP currency account.

9. Recovery: Batch Timeout

Simulating a network failure during a "Close Batch" operation to ensure the system doesn't leave the batch in a 'Locked' state.

10. Database Schema Evolution

Testing application stability after a MongoDB schema update where optional fields are missing in legacy records.
